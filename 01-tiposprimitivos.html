Tipos Primitivos

O que são Tipos Primitivos?

Tipos Primitivos são todos os elementos com os quais o JavaScript trabalha, assim como em html trabalhamos com números, caracteres, objetos e etc, no JavaScript é a mesma coisa com algumas diferenças.

É importantíssimo entender esse conceito, pois entender a diferença entre os tipos primitivos é o que vai nos permitir usar bem as nossas variáveis. Assim como vimos o exemplo de um estacionamento no artigo sobre variáveis, o mesmo conceito se vale aqui quando falamos de Tipos Primitivos. Assim como um caminhão tem 6 rodas e não pode caber na vaga de uma moto, o mesmo se dá com os Tipo Primitivos, eles são formados de características diferentes, e um Tipo Primitivo Number não pode ocupar uma variável reservada a uma String a menos que a conversão correta dos tipos seja feita.

Quais são os Tipos Primitivos?

  • Number - Esse é o tipo primitivo que representa os números, como por exemplo o conjunto de números Reais (1, 2, 3..) e o conjunto de números Inteiros (1,5, 2,5, 3,5...) ou de Ponto Flutuante, chamamos de ponto flutuante por que para serem reconhecidos pelo JavaScript é necessário o uso do "ponto" ao invés da "vírgula". (1.5, 2.5, 3.5...) Dentro da cadeia dos tipos primitivos Number encontramos também 2 tipos primitivos específicos, os: Infinity e o NaN, que falaremos melhor em outro artigo;
  • String - Esse é o tipo primitivo que representa os caracteres, como: letras, ideogramas e até mesmo números representativos, como o CPF, RG, CNH e etc. Para representarmos os strings nós sempre usamos áspas duplas ou simples "" '' e a crase ``. Em geral o uso de áspas duplas ou simples não faz diferença quando escrevemos caracteres em JavaScript, mas o uso da crase faz uma grande diferença, pois usamos a crase como um "template string", ou "formatação string", que nada mais é do que quando nós desejamos criar uma string que irá concatenar as palavras com outros elementos de fora;
  • Boolean - Esse é o tipo primitivo que representa as portas booleanas de George Boole, as famosas: true e false, entre muitas outras. Falaremos mais a respeito delas em outro artigo;
  • Null - Esse é o tipo primitivo que representa os elementos nulos, cuidado para não confundir com os Undefineds, pois os tipos nulos são encarados pelo JavaScript como um objeto;
  • Undefined - Esse é o tipo primitivo que representa os elementos indenidos, algumas pessoas confundem com os tipos nulos, mas o Undefined é um elemento sem definição enquanto o null é encarado pelo JavaScript como um Object;
  • Object - Esse é o tipo primitivo que representa os Objetos, para representarmos objetos dentro do JavaScript basta nós utilizarmos as chaves { }. Dentro dos Object temos um outro tipo primitivo incluso, que é o array, mais conhecido como vetor que também são enxergados pelo JavaScript como objetos e podem ser representados pelos sinais de colchetes [ ];
  • Function - Esse é o tipo primitivo que representa as Funções em JavaScript, como as fuctions funcionam dentro do paradigma funcional do JavaScript, elas também são consideradas como Tipos Primitivos;
