Operadores utilizados em JavaScript

Operadores

Antes de ser uma simples ferramenta de interação entre um site e seus usuários, o JavaScript é uma ferramenta de fazer contas, portas lógicas e muitos outros tipos de operações matemáticas. Afinal, estamos falando de uma linguagem de programação para computadores, e computadores na sua essência não passam de calculadoras super potentes capazes de usar a matemática para resolver questões complexas.

Que tipo de operações matemáticas o JavaScript é capaz de fazer?

O JavaScript é capaz de fazer operações:

  • Aritméticas - usando a operadores aritméticos que já conhecemos;

  • Atribuições - usando o valor de um operador e atribuindo-o sobre outro operador;

  • Relacionais - para estabelecer relações entre 2 ou mais valores, que resultará na aplicação de um valor booleano (true ou false);

  • Lógicos - para fazer operações sobre portas lógicas;

  • Ternários - para fazer operações condicionais;

  • Typeof - para indicar tipos primitivos;

Operadores Aritméticos

Os operadores aritméticos que nós utilizamos no JavaScript são:

  • Soma - Utilizamos por através do sinal de " + ";
    Ex: 5 + 2 = 7
    (Nessa questão nós devemos tomar cuidado, pois o sinal de " + " também é usado para concatenar strings, se os nossos valores não estiverem devidamente identificados como numbers, a soma não será feita da maneira correta.)

  • Subtração - Utilizamos por através do sinal de " - ";
    Ex: 5 - 2 = 3

  • Multiplicação - Utilizamos por através do sinal de " * ";
    Ex: 5 * 2 = 10

  • Divisão Real- Utilizamos por através do sinal de " / ";
    Ex: 5 / 2 = 2.5

  • Divisão Inteira - Utilizamos por através do sinal de " % ";
    Ex: 5 % 2 = 1
    (O valor final desse operador sempre será o resultado do resto de uma divisão de números inteiros.

  • Potência - Utilizamos por através do sinal de " ** ";
    Ex: 5 ** 2 = 25
    (Onde um número elevado sobre uma determinada potência sempre será multiplicado por ele mesmo o número de vezes que a sua potência exigir.)

Precedência de Operadores Aritméticos

Os operadores aritméticos tem de obedecer a uma ordem de precedêcia, entender isso é importante para criarmos comandos corretos no JavaScript. Como funciona essa ordem de precedência?

  • 1º - Tudo oque estiver entre ( ) "parênteses";
    Ex: (5 + 3) / 2
            8 / 2 = 4

  • 2º - As operações de ** "potência";
    Ex: 5 + 3 ** 2
          5 + 9 = 14

  • 3º - As operações de * "multiplicação" / "divisão real" % "divisão inteira"; (Quando nós temos dois operadores da mesma ordem de precedência em uma operação, fazemos primeiro aquela que vem primeiro, por exemplo: 5 * 3 / 2, a primeira operação a ser feita nesse caso seria 5 * 3, e só depois nós faríamos a / 2.
    Ex: 5 + 3 * 2
          5 + 6 = 11

    Ex: 5 + 3 / 2
          5 + 1.5 = 6.5

    Ex: 5 + 3 % 2 = 6
          5 + 1 = 6

  • 4º - As operações de + "soma" - "subtração";
    Ex: 5 + 3 * 2
          5 + 6 = 11

    Ex: 5 - 3 * 2
          5 - 6 = 1

Operadores de Atribuição

Operadores de Atribuição são todos aqueles onde nós atribuímos um determinado valor á uma variável, ou modificamos o valor que uma variável já possuí utilizando outras formas de operação.

Entre os operadores de Atribuição temos os seguintes modelos:

  • Atribuição Simples
    É onde nós fazemos simples atribuições a uma variável, ou atribuímos uma variável dentro de outra variável, como nos exemplos abaixo:
    var a = 5 + 3    =>    var a = 8
    var b = a % 5    =>    var b = 3
    var c = 5 * b ** 2    =>    var c = 5 * 9    =>    var c = 45
    var d = 10 - a / 2    =>    var d = 10 - 4    =>    var d = 6
    var e = 6 * 2 / d    =>    var e = 12 / d    =>    var e = 2
    var f = b % e + 4 / e    =>    var f = 1 + 2    =>    var f = 3

  • Auto-Atribuição
    É quando nós mudamos o valor de uma variável atribuíndo o valor dela sobre si mesma acrescentando algum tipo de operação, como nos exemplos abaixo:
    var n = 3
    var n = n + 4     =>    var n = 7
    var n = n - 5     =>    var n = 2
    var n = n * 4     =>    var n = 8
    var n = n / 2     =>    var n = 4
    var n = n ** 2     =>    var n = 16
    var n = n % 5     =>    var n = 1
    (No exemplo acima vimos auto-atribuições na variável "n" em todos os exemplos de operadores aritméticos, onde o valor da variável "n" começa com "3" e termina com "1".)

  • Auto-Atribuição Simplificada
    É quando nós mudamos o valor de uma variável atribuíndo o valor dela sobre si mesma acrescentando algum tipo de operação, porém com a vantagem de ser mais simplificada do que a auto-atribuição simples,como nos exemplos abaixo:

    var n = n + 2;      poderíamos trocar por:   var n += 2;
    var n = n - 2;      poderíamos trocar por:   var n -= 2;
    var n = n * 2;      poderíamos trocar por:   var n *= 2;
    var n = n / 2;      poderíamos trocar por:   var n /= 2;
    var n = n ** 2;      poderíamos trocar por:   var n **= 2;
    var n = n % 2;      poderíamos trocar por:   var n %= 2;

  • Auto-Incremento e Auto-Decremento

    Entre os operadores de atribuição temos também o "incremento" e o "decremento", que nada mais são do que auto-atribuições simplificadas onde desejamos acrescentar ou retirar apenas o valor de "1". Por estarem num formato ainda mais simples que as auto-atribuições simplificadas, podemos ganhar muito tempo durante a escrita de linhas de código. Vejamos alguns exemplos abaixo:


    Incremento

    Em vez de usarmos: var n = n + 1     usaríamos: var n++ e pronto!!
    (Automáticamente o valor de "1" será acrescentado! Lembrando que o auto-incremento só pode adicionar o valor de "1", se fizermos "n++2" ou "n+++++", vai dar erro!!)

    Decremento

    Em vez de usarmos: var n = n - 1     usaríamos: var n-- e pronto!!
    (Automáticamente o valor de "1" será subtraído! Lembrando que o auto-decremento só pode subtrair o valor de "1", se fizermos "n--2" ou "n-----", vai dar erro!!)

  • Pré-Incremento e Pré-decremento

     As atribuições de incremento e decremento adicionam ou subtraem o valor de "1" de uma variável durante uma operação. Existem uma forma simplificada de nós adicionarmos ou subtrairmos o valor de "1" antes de uma operação acontecer, que é usando o pré-incremento e o pré-decremento. Podemos fazer isso das seguintes maneiras:

    Pré-Incremento

    Em vez de usarmos: var n++     usaríamos: var ++n e pronto!!
    (O valor de "1" será somado ao valor da variável antes da operação.)

    Pré-Decremento

    Em vez de usarmos: var n--     usaríamos: var --n e pronto!!
    (O valor de "1" será subtraído do valor da variável antes da operação.)

Operadores relacionais

Operadores relacionais são aqueles que estabelecem algum tipo de relação entre dois valores ou mais gerando algum tipo de resposta booleana, ou seja true (verdadeiro) ou (false). (É imporatante lembrar que em uma operação que existir operadores aritméticos e operadores relacionais, os operadores aritméticos tem a prioridade na ordem de precedência.)

Vejamos alguns tipos de operadores relacionais:


  • " > " (maior)

    Este é o operador "maior", ele estabelece uma relação entre dois valores e diz se um valor realmente é maior do que o outro valor determinado, isso vai gerar um resultado booleano, dizendo se a nossa afirmação é true ou false. Por exemplo:

    5 > 2 = true


  • " < " (menor)

    Este é o operador "menor", ele estabelece uma relação entre dois valores e diz se um valor realmente é menor do que o outro valor determinado, isso vai gerar um resultado booleano, dizendo se a nossa afirmação é true ou false. Por exemplo:

    7 < 4 = false


  • " >= " (maior ou igual)

    Este é o operador "maior ou igual", ele estabelece uma relação entre dois valores e diz se um valor realmente é maior ou igual ao outro valor determinado, isso vai gerar um resultado booleano, dizendo se a nossa afirmação é true ou false. Por exemplo:

    8 >= 8 = true


  • " <= " (menor ou igual)

    Este é o operador "menor ou igual", ele estabelece uma relação entre dois valores e diz se um valor realmente é menor ou igual ao outro valor determinado, isso vai gerar um resultado booleano, dizendo se a nossa afirmação é true ou false. Por exemplo:

    9 <= 7 = false


  • " == " (igual)

    Este é o operador "igual", note que no JavaScript o sinal de "igual" é formado por dois sinais de igual da matemática comum, isso acontece por que no JavaScript, assim como em outras linguagens de programação, o sinal de igual único recebe o nome de "recebe", ele é encarado como um operador de atribuição não como sinal de igual. Portanto, como forma de diferenciar o sinal de "recebe" do sinal de "igual", nós utilizamos o igual duplo (" == "). Esse operador estabelece uma relação entre dois valores e diz se um valor realmente é igual ao outro valor determinado, isso vai gerar um resultado booleano, dizendo se a nossa afirmação é true ou false. Por exemplo:

    5 == 5 = true


  • " != " (diferente)

    Este é o operador "diferente", ele é indicado assim por que o sinal de " ! " é encarado como símbolo de negação nas linguagens de programação, fazendo com que um valor positivo se torne negativo, e um valor verdadeiro se torne falso. Como esse símbolo está vindo antes do sinal de igual, ele seria como uma espécie de negação do valor de igual, ou seja, tudo áquilo que não é igual só pode ser diferente, e daí vem o seu nome "diferente". Esse operador estabelece uma relação entre dois valores e diz se um valor realmente é diferente do que o outro valor determinado, isso vai gerar um resultado booleano, dizendo se a nossa afirmação é true ou false. Por exemplo:

    4 != 4 = false

  • " === " (igual com restrições)

    Este é o operador "igualdade restrita" que, diferente do operardor de igual, não tem apenas dois sinais de igual da matemática comum, mas sim "3 sinais de igual", mesmo sendo diferente ele ainda é um sinal de igual, porém com restrições, como assim? Bom, se você perguntar ao JavaScript se 5 == '5' (Lembrando que tudo o que está entre áspas é encarado como uma string.) o JavaScript vai dizer que estes valores são "true", apesar de serem de tipos primitivos diferentes. Por que isso acontece? Acontece por que os valores 5 e '5' são de grandezas iguais apesar de serem de tipos diferentes. Para solucionar esse impasse, foi criado o operador " === " ou "igual restrito" que não permite que 2 valores, apesar de serem de grandezas iguais sejam encarados como iguais por serem de tipos diferentes, gerando um resultado booleano, dizendo se a nossa afirmação é true ou false. Por exemplo:

    4 === '4' = false

  • " !== " (diferente com restrições)

    Este é o operador "desigualdade restrita" que, diferente do operardor de diferente, não tem apenas 1 sinail de igual seguido do sinal de negação, mas sim "2 sinais de igual", mesmo sendo diferente ele ainda é um sinal de desigualdade, porém com restrições, como assim? Bom, se você perguntar ao JavaScript se 5 != '5' (Lembrando que tudo o que está entre áspas é encarado como uma string.) o JavaScript vai dizer que essa relação é "false", apesar de serem de tipos primitivos diferentes. Por que isso acontece? Acontece por que os valores 5 e '5' são de grandezas iguais apesar de serem de tipos diferentes. Para solucionar esse impasse, foi criado o operador " !== " ou "desigualdade restrita" que não permite que 2 valores, apesar de serem de grandezas iguais sejam encarados como iguais por serem de tipos diferentes, gerando um resultado booleano, dizendo se a nossa afirmação é true ou false. Por exemplo:

    4 !== '4' = true

Observações:

Entre os operadores relacionais não existe uma ordem de precedência, o operador que vier primeiro em uma expressão, será aquele que vai ser feito.

Operadores Lógicos

Os operadores Lógicos são aqueles que fazem uso das portas lógicas booleanas não somente para fazer comparações entre variáveis, mas para fazer comparações entre operações relacionais inteiras gerando valores de true ou false.

(Lembrando que, na ordem de precedêcia entre os tipos de operadores, os operadores lógicos são aqueles que vêm em 3º lugar, perdendo para os operadores aritméticos que vem em 1º lugar e para os operadores relacionais que vem em 2º lugar.)

Vejamos agora alguns exemplos de operadores lógicos


  • " ! " (negação)
    Esse é o valor de "negação" no JavaScript, assim como também em outras linguagens de programação. E ele faz com que todo valor seja negado, ou seja, invertido para um valor contrário, ele é a famosa porta booleana "NOT". Nós o utilizamos da seguinte maneira:

    var = ! 2 * 8 > 20
    var = ! 16 > 20
    var = ! false
    var = true

    (Como você pode ver no exemplo acima, o valor que deveria gerar um false, por que 16 não é > (maior) que 20, foi invertido para um true, graças ao operador de negação.)


  • " && " (conjunção)
    Esse é o valor de "conjunção", no JavaScript, assim como também em outras linguagens de programação, esse operador faz a comparação entre duas operações relacionais e faz o papel da famosa porta booleana AND.
    Na porta AND é feita uma comparação entre dois valores que podem gerar true ou false, o operador AND só irá retornar o valor true se ambos os valores das operações também tiverem resultado true, qualquer outra comparação só pode gerar valor false. O exemplo a seguir exemplifica bem os valores possíveis de serem gerados em um operador &&:

    true && true = true
    true && false = false
    false && true = false
    false && false = false

    Nós o utilizamos da seguinte maneira:

    var = 2 * 8 > 20 && 4 * 6 <= 24
    var = 16 > 20 && 24 <= 24
    var = false && true
    var = false

    (Como você pode ver no exemplo acima, o valor gerado na conjunção resultou num false, por que as duas operações relacionais não geraram um true. Para o resultado ser true, seria necessário que as duas operações gerassem true, e não apenas uma delas. É como se fizéssemos uma comparação onde você me pede duas canetas, uma azul e (AND) uma vermelha, eu tenho que lhe entregar as 2 para que você fique satisfeito. Na porta lógica AND se dá o mesmo, a porta AND só ficará satisfeita e retornará o valor como "true", se os 2 valores nas operações relacionais também gerarem um "true", qualquer outro resultado iria gerar um "false" na porta lógica.)


  • " || " (disjunção)

    Esse é o valor de "disjunção", no JavaScript, assim como também em outras linguagens de programação, esse operador faz a comparação entre duas operações relacionais e faz o papel da porta booleana OR.
    Na porta OR é feita uma comparação entre dois valores que podem gerar true ou false, o operador OR retorna o valor true se qualquer um dos valores nas operações relacionais resultar em true. Caso contrário, se ambos os valores resultarem em false, o resultado da operação também será false. Podemos entender isso bem analisando todos os resultados possíveis de uma operação de disjunção no exemplo abaixo:

    true || true = true
    true || false = true
    false || true = true
    false || false = false

    Nós o utilizamos da seguinte maneira:

    var = 2 * 8 > 20 || 4 * 6 <= 24
    var = 16 > 20 || 24 <= 24
    var = false || true
    var = true

    (Como você pode ver no exemplo acima, o valor gerado na disjunção resultou num true, por que ao menos 1 das operações relacionais gerou um true. Para que o resultado fosse false, seria necessário que as duas operações gerassem false, e não apenas uma delas. É como se fizéssemos uma comparação onde você me pede uma caneta, ela pode ser azul ou (OR) vermelha, tanto faz a cor, se eu lhe der ao menos uma delas, você já estará satisfeito. Na porta lógica OR se dá o mesmo, desde que ao menos um dos valores gere um "true", o porta OR já está satisfeita e retornará o valor como "true".)

Ordem de Precedência dos Operadores Lógicos

Vale lembrar que, quando estamos falando de operadores lógicos em uma única expressão, a ordem de precedência entre eles é a seguinte:

  • 1º Lugar - Operador de Negação ( ! );
  • 2º Lugar - Operador de Conjunção ( && );
  • 1º Lugar - Operador de Disjunção ( || );

Operador Ternário

O operador ternário recebe esse nome por que ele é um único operador que faz a função de três operadores. O operador ternário tem a função de retornar o valor de uma determinada operação lógica ou relacional gerando um valor diferente caso o resultado seja true ou false.

(Lembrando que, na ordem de precedêcia entre os tipos de operadores, o operador ternário está em último lugar, perdendo para os operadores aritméticos que vem em 1º lugar, para os operadores relacionais que vem em 2º lugar e para os operadores lógicos que vêm em 3º lugar)

A sintáxe do operador ternário é a seguinte:

                  " ? : "

onde colocamos os valores de:
teste lógico ? valor se "true" : valor se "false"

Vejamos como utilizar esse operador em um caso real:

Digamos que uma determinada variável "idade" receba o valor de "17"

var idade = 17

E digamos que desejamos saber se a pessoa que atribuíu a sua idade a variável "idade" já é maior de idade ou não. Para isso usaremos o operador ternário " ? : "

var idade = 17
idade >= 18 ? 'maior' : 'menor';
var idade = 'menor';

(Perceba que colocamos o valor do teste lógico (idade >= 18) e o valor da reposta "true" ('maior') entre o sinal de interrogação " ? ". E colocamos o valor da resposta "false" ('menor') depois do sinal de " : "; Isso quer dizer que, caso a pessoa tenha a idade inferior a 18 anos, o operador ternário irá gerar a resposta 'menor', se ela tiver a idade igual ou superior a 18 anos, o operador irá gerar a resposta 'maior'.)

Relação de Precedência de Operadores

A relação de precedência dos operadores se dá da seguinte maneira:

  • 1º Lugar - Tudo o que estiver dentro de Parênteses   ( ) ;

  • 2º Lugar - Operadores Aritméticos   **   /   %   *   +   -   ;

  • 3º Lugar - Operadores Relacionais   >   <   >=   <=   ==   !=   ===   !==   ;

  • 4º Lugar - Operadores Lógicos   !   &&     ||   ;

  • 5º Lugar - Operador Ternário   ? :   ;
